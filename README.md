# Название проекта

Список сотрудников



## Описание
Проект выполнен согласно предоставленного технического задания. Разработана база данных и web-приложение 
для ведения списка сотрудников, с указанием их должностей. Приложение поддерживает следующий функционал:

      - Отображение списка должностей. Поля "Должность", "Категория" (выбор из вариантов: Руководитель,
        Специалист, Служащий, Рабочий);

      - Добавление, удаление, изменение должностей через интерфейс приложения;

      - Отображение списка сотрудников. Поля "ФИО", "Пол", "Возраст", "Должность", "Категория";

      - Добавление, удаление сотрудника;

      - Просмотр и редактирование личной карточки сотрудника. Поля: "ФИО", "Пол", "Возраст". 
        Выбор должности из списка.

Проект размещён в git-репозитории находящемся по адресу: https://github.com/slavaz950/List_Employ.git. Текстовый 
файл SQL-скрипта находится в корне этого репозитория (sql_data.sql). SQL-скрипт интегрирован в Bash-скрипт deploy.sh, 
который также находится там же. Скрипт deploy.sh отвечает за полное развёрьывание проекта на целевом компьютере с 
предустановленными Astra Linux 1.6 или 1.7. Скрипт автоматически определяет все параметры развёртывания в зависимости 
от версии операционной системы согласно технического задания. После выполнения файла deploy.sh пользователь может 
набрав в адресной строке браузера адрес: http://localhost:8000  приступать к работе с данной информационной системой.

В процессе тестирования был замечен небольшой нюанс. В браузере Firefox категорически отказывались работать put-запросы. 
Ошибок никаких не было. Но и изменений в базе данных не происходило. При том что добавление новых сотрудников и 
должностей происходило штатно. В тоже время в браузерах Chromiun  и Chromium-gost весь функционал в полном объёме работает. 
Предполагаю что возможная причина в настройках безопасности браузера или самой операционной системы. И напрямую к 
функциональности систимы это баг не относится.




## Установка

Перед началом развёртывания проекта необходимо убедиться в доступности стандартных репозиториев Astra Linux. Иначе при 
выполнении скрипта могут возникнуть ошибки.

Для развёртывания проекта на целевом компьютере, необходимо выполнить три действия:

1.  Любым удобным способом скопируйте файл deploy.sh на целевой компьютер и сделать этот файл исполняемым. При 
    этом директория куда Вы сохраните этот файл не имеет никакого значения. Все пути по которым будет разворачиваться 
    проект уже прописаны внутри скрипта. Проект будет развёрнут в директории /opt/List_Employ.

2. Запустите файл deploy.sh и дождитесь окончания выполнения всех операций.

3. Откройте браузер и введите в браузерной строке адрес: http://127.0.0.1:8000  или  http://localhost:8000




## Использование

На панели управления в самом верху страницы находятся два пункта меню: "Сотрудники" и "Должности". Каждый из этих 
пунктов содержит в себе выпадающее меню, внутри которого находятся по два пункта. Один пункт для вывода списка, а 
второй для добавления нового элемента в зависимости от контекста (либо сотрудник, либо должность).

Переходя к списку сотрудников пользователь видит всех сотрудников добавленных в базу данных. Напротив каждого 
сотрудника находится по три кнопки, которые при необходимости позволяют выполнять над текущим сотрудником одно из 
представленных действий (Просмотр карточки сотрудника, Изменение сотрудника, Удаление сотрудника). Со страницы 
карточки сотрудника пользователь также может перейти к Изменению сотрудника.

Переходя к списку должностей пользователь видит все должности добавленные в базу данных в соответствии с выбранной в 
поле "Категория" категории. В остальном работа с данным справочником ничем не отличается от работы с сотрудниками. За 
одним лишь исключением, что здесь нет карточки должностей по аналогии с сотрудниками.


Для того чтобы информационная система была в работоспособном состоянии, всегда должен быть запущен локальный сервер.



ЗАПУСК ЛОКАЛЬНОГО СЕРВЕРА

Запуск локального сервера в консоли (посредством набора команд)

cd /opt/List_Employ
source venv/bin/activate
python manage.py runserver 127.0.0.1:8000


Запуск локального сервера через скрипт (start.sh)
Файл start.sh находится в корне проекта. Файл можно запускать из любого удобного для пользователя места на компьютере. 
Важно чтоб файл этот был исполняемым.


ОСТАНОВКА СЕРВЕРА
Для остановки сервера нажмите Ctrl+C
